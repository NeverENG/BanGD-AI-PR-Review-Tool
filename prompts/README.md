# prompts/ — BanGD 的核心智力资产

这里的内容是 BanGD 评审质量的来源（占效果约 80%），比管道代码更重要。将来核心评审模块会把这些文件加载进 Claude 的系统提示词，并对这些大块固定内容开启 **prompt caching**。

| 文件 | 作用 |
|---|---|
| `system-prompt.md` | BanGD 的人设、与通用评审者的区别、工作方式、输出格式 |
| `rubric.md` | 数据库内核评审清单——每个维度该追问什么、往哪个架构方向推 |
| `examples/` | few-shot 范例，锚定评审的深度与中文输出风格 |

## 迭代约定

- 这是**活文档**。随着评测语料集（见 CLAUDE.md 的 open questions）积累，根据"BanGD 建议 vs 专家评审"的差距持续修订 rubric 和范例。
- 新增范例时**保持维度多样性**，不要全是"双表设计"——架构级方案因问题而异（见 `wal-ordering-crash-safety.md`），过拟合到单一套路会降低评审质量。
- 每条 finding 的输出结构固定为：`问题根因 → 为什么低级解法不够 → 架构级方案 → 代价/收益`。
